## 2. 전달 방법 (Codex는 이 폴더를 어떻게 받는가)

Codex CLI는 현재 작업 디렉토리의 `AGENTS.md`를 **자동으로** 세션 컨텍스트에 포함한다. 그래서 전달 방법은 단 하나:

**이 `codex-handoff/` 폴더의 내용물을 Codex가 실행될 프로젝트 루트로 복사한다.**

즉 서버W WSL에서:

```bash
# WSL Ubuntu 터미널
cd ~/projects
mkdir govpress-mcp && cd govpress-mcp
git init -b main

# 이 폴더의 내용을 프로젝트 루트로 복사
HANDOFF="/mnt/c/Users/wavel/OneDrive/문서/Claude/Projects/보도자료 PDF-MD 변환/codex-handoff"

cp "$HANDOFF/AGENTS.md" .                        # Codex가 자동 인식
mkdir -p src/govpress_mcp prompts tests
cp "$HANDOFF/bulk_ingest.py" src/govpress_mcp/          # Codex가 여기 TODO 채움
cp "$HANDOFF/prompts/bulk_ingest_phase1.md" prompts/
```

이 시점에 `~/projects/govpress-mcp/` 구조:

```
govpress-mcp/
├── AGENTS.md                        ← Codex 자동 로드
├── prompts/
│   └── bulk_ingest_phase1.md        ← 네가 Codex에 복붙할 지시
├── src/govpress_mcp/
│   └── bulk_ingest.py               ← Codex가 완성할 파일
└── (이후 Codex가 만드는 파일들)
```


### 3.3 Codex가 작업 중일 때 네가 할 것

Codex가 스스로 돌아가는 동안 다음 4개 지점에서 한 번씩만 개입:

1. **submodule 추가 직후** — `git status`로 `vendor/gov-md-converter` 제대로 등록됐는지
2. **vendored/policy_briefing.py 복사 직후** — 파일 상단 SHA 주석 있는지
3. **스모크 10건 완료 후** — `data/md/2026/04/*.md` 하나 열어서 frontmatter 눈으로 확인, 서비스키 유출 없는지
4. **재실행 후** — 로그에 `skip-unchanged` 10건 정확히 찍히는지

각 지점에서 이상 있으면 Codex에게 바로 지적:

```
잠깐. {상황 설명}. AGENTS.md §{N}을 다시 봐라. 수정해라.
```

## 4. Codex가 "멈추고 보고" 했을 때 — 체크포인트 3개

Phase 1은 한 번에 끝나지 않는다. Codex는 **M1 → 멈춤 → 사람 승인 → M2 → 멈춤 → 사람 승인 → M3 → 끝** 순서. AGENTS.md §4에 정의돼 있다.

### 4.1 M1 완료 시 (토 오전) — MD 수작업 검수

Codex가 `M1 완료. 10건 스모크 성공. 승인 대기.` 같은 메시지로 멈추면:

```bash
cd ~/projects/govpress-mcp

# 기본 카운트 체크
ls data/raw/2026/04/ | wc -l             # == 10
ls data/md/2026/04/ | wc -l              # == 10
sqlite3 data/fetch-log/checksums.db "SELECT COUNT(*) FROM checksums"  # == 10

# tests 확인
pytest -q                                 # 3개 pass

# 서비스키 전수 grep (중요)
grep -rn "$GOVPRESS_POLICY_BRIEFING_SERVICE_KEY" . 2>/dev/null | grep -v ".env"
# → 아무것도 안 나와야 함

# api2.govpress.cloud 호출 0건
grep -r "api2.govpress.cloud" logs/ 2>/dev/null | grep -v "FORBIDDEN_HOSTS"
# → 아무것도 안 나와야 함 (FORBIDDEN 차단 로그는 OK)

# MD 1~2개 실제 내용 열어보기 — frontmatter가 규약대로인지 사람 눈으로
head -30 data/md/2026/04/*.md | head -60
```

문제 없으면 Codex 창에 **"M2 진행"** 이라고만 입력. 문제 있으면:

```
잠깐. {파일명}:{줄번호}에 {문제}. AGENTS.md §{N}에 위배.
수정하고 다시 M1 보고해라.
```

### 4.2 M2 완료 시 (토 오후~일 오전) — 리허설 수치 검토

Codex가 `M2 완료. 1개월 리허설 성공.` 로 멈추면:

```bash
cat docs/rehearsal-report.md

# 디스크 증가 실측
du -sh data/raw/2026/03/ data/md/2026/03/

# 실패가 어디서 몰렸는지 분포
cat data/fetch-log/failed.jsonl 2>/dev/null | jq -r '.reason' | sort | uniq -c
```

AGENTS.md §4.2의 5개 조건이 모두 충족됐으면 **"M3 진행"** 지시.
충족 못 한 게 있으면 — 특히 성공률 <95%, HWP 구버전 >10% 면 Claude 세션으로 돌아와 원인 분석 요청.

### 4.3 M3 완료 시 (평일, 2~4주 후) — Phase 1 진짜 종료

Codex가 `M3 완료. Phase 1 종료.` 로 멈추면:

```bash
cat docs/phase1-report.md                # 전체 리포트
cat LICENSE-data                          # 공공누리 고지 확인
systemctl --user status govpress-mcp-daily.timer # 일일 증분 timer 살아있는지

git log --oneline | head -20              # 전체 커밋 흐름
```

이 시점에 Claude 세션(여기)으로 돌아와 **"Codex가 Phase 1 M3 종료. phase1-report.md와 git log 공유할게. Phase 2 설계 진행"** 이라고 말한다. `docs/phase1-report.md` 내용을 붙여넣으면 된다.

### 4.4 비상 호출 (EMERGENCY STOP) 시

Codex가 마일스톤 중간에 `EMERGENCY STOP: ...` 로 멈추면 자동 재시도하지 말 것. Claude 세션에서 Codex의 보고 메시지 그대로 공유하고 대응 지시를 받는다. 특히 아래 두 가지는 즉시 대응:

- **서비스키 노출** → 키 즉시 rotation (data.go.kr에서 재발급), 커밋 히스토리 `git filter-repo`로 제거
- **api2.govpress.cloud 호출** → 훅이 왜 작동 안 했는지 재조사, 회귀 테스트 추가

## 5. Codex가 막혔을 때 대응 표

| 증상 | 원인 가능성 | 대응 |
|---|---|---|
| `PolicyBriefingClient` import 실패 | vendored 복사 경로 오류 | `src/govpress_mcp/vendored/__init__.py` 있는지, PYTHONPATH 확인 |
| `govpress_converter.convert_hwpx` 에러 | wheel 설치 안 됨 | `pip install -e vendor/gov-md-converter` 재실행 |
| 모든 item이 `is_zip_container=False` | 특정 날짜가 전부 HWP 구버전 | 다른 날짜로 재시도 (2026-04-10 → 04-15) |
| 429 폭주 | 동시성 5 초과 | Codex에 "semaphore=3으로 내리고 sleep 0.5초로 올려라" |
| `urlopen` 차단 | FORBIDDEN_HOSTS 훅이 korea.kr까지 막음 | 훅이 hostname 매칭 시 endswith 대신 exact 비교하는지 확인 |
| 서비스키 로그 노출 발견 | `LOG.info(f"url={url}")` 류 | Codex에 "키를 masking하고 커밋 히스토리에서도 제거(`git filter-repo`)" |
