# HWP → HWPX 변환 결과 보고서 (2026-04-20)

## 요약

- 작업 대상: `data/raw` 내 레거시 HWP `33,853`건
- 서버H(Windows + 한/글 COM)에서 `.hwp -> .hwpx` 변환 수행
- 서버W `data/raw`로 성공 산출물 업로드 완료
- 최종 결과:
  - 변환 성공 `33,801`건
  - 변환 제외 `52`건

## 최종 판정

| 항목 | 건수 |
|---|---:|
| 대상 HWP | 33,853 |
| 자동/보정 포함 성공 HWPX | 33,801 |
| 최종 제외 | 52 |

성공률은 `99.85%` (`33,801 / 33,853`)이다.

## 수행 메모

- 서버H에서 `pywin32`와 `HWPFrame.HwpObject` COM 동작을 확인한 뒤 변환 스크립트를 실행했다.
- 일부 환경에서 한/글 파일 접근 보안 대화상자가 떠서 자동 허용 보조 래퍼를 사용했다.
- 대량 배치 중 일부 파일은 `RPC 서버를 사용할 수 없습니다` 오류가 발생해 별도 재시도를 수행했다.
- 재시도 과정에서 일부 파일은 콘솔상 성공으로 보이지만 실제 `.hwpx` 산출물이 남지 않는 사례가 확인되어, 스크립트에 `SaveAs` 후 실제 출력 파일 존재 검증을 추가했다.

## 제외 52건

남은 `52`건은 한/글에서 직접 열었을 때 `배포용 문서`로 표시되었고, 다른 형식으로 저장이 불가능했다. 따라서 이 집합은 HWPX 변환 불가 예외로 확정했다.

수동 확인용 원본 보관 경로:

- `C:\temp\hwp_manual_retry`
- `C:\temp\hwpx_manual_retry`

작업 메모:

- `README-manual-failures.txt`에 제외 사유를 기록했다.
- 원본 `.hwp`는 삭제하지 않고 유지한다.

## 운영상 의미

이 결과를 기준으로 HWP 레거시 큐는 다음처럼 해석한다.

- 대부분의 HWP 레거시 문서는 서버H 수동/반자동 변환으로 해소 가능했다.
- `배포용 문서` 52건은 현재 한/글 저장 정책상 HWPX 변환 경로가 없으므로 별도 예외 집합으로 관리해야 한다.
- 이후 재처리 파이프라인은 성공 산출물 `33,801`건만 대상으로 HWPX → Markdown 변환을 진행하면 된다.
